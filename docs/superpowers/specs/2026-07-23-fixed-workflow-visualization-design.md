# 固定工作流可视化产品设计

日期：2026-07-23
状态：已完成产品讨论，待用户审阅文档

## 1. 背景与结论

Stratum 已具备固定 DAG 工作流的核心执行内核，包括草稿、不可变发布版本、Agent/Skill/MCP/条件/审批节点、并行调度、重试、超时、幂等启动、暂停、恢复、取消、审批、人工介入、持久化事件和 SSE 查询。

当前后端仍是按已知 ID 操作的能力接口，缺少工作流目录、版本列表、运行历史、运行发起人归属、结构化输入契约和面向运行中的输出增量。因此，V1 不是单纯增加前端页面，而是为现有执行内核补齐最小产品契约，并交付管理员配置、终端用户执行和运行治理的完整闭环。

## 2. 产品目标

- 管理员不编辑 JSON 即可创建、验证和发布固定 DAG 工作流。
- 普通用户从工作流中心或 Agent 对话快捷入口启动已发布工作流。
- 用户以自然语言主任务加结构化附加字段提供输入。
- 用户实时看到 DAG 进度、当前节点、输出、工具步骤、等待点和失败位置。
- 普通用户只访问自己的运行；管理员治理租户内全部运行。
- 保持确定性外层 DAG 与节点内部 Agent 决策的职责边界。

## 3. 用户与权限

### 3.1 普通用户

- 浏览租户内已发布工作流。
- 查看工作流说明和当前发布版本的输入要求。
- 启动工作流。
- 查看和取消自己发起的运行。
- 不可查看其他用户的运行。
- 不可编辑、验证、发布工作流或处理审批与人工介入。

### 3.2 管理员

- 查看租户内全部草稿和已发布工作流。
- 创建、编辑、验证和发布工作流。
- 查看不可变历史版本。
- 查看和控制租户内全部运行。
- 处理审批和人工介入。

对象级授权必须在 application 层执行。前端可根据权限调整入口，但不能作为授权边界。

## 4. 信息架构

### 4.1 工作流中心

所有成员可进入。页面展示已发布工作流的名称、说明、版本、最近更新时间和用户最近一次运行状态。列表不超过五列，支持名称搜索和分页。空状态为“工作流还是空的”，搜索无结果为“没有找到匹配的工作流”。

普通用户进入工作流详情后可填写输入并启动。管理员额外看到“编辑草稿”入口。

### 4.2 工作流设计器

仅管理员可进入。采用引导式画布，而非原始 JSON 或无限自由的工程画布：

- 左侧：Agent、Skill、MCP 工具、条件、人工审批节点和租户内业务模板。
- 中间：真实 DAG 画布，支持插入步骤、连线、选择、删除、缩放和适应画布。
- 右侧：所选节点的基础配置；输入输出映射、超时和重试折叠到高级设置。
- 底部：结构化验证结果，错误可定位并聚焦到节点或连线。
- 顶部：保存草稿、验证、发布，保持明确的操作顺序。

V1 使用显式保存，不做自动保存或离线编辑。保存携带预期修订号，冲突时保留本地内容并阻止静默覆盖。

### 4.3 运行中心

普通用户默认看到“我的运行”；管理员可以查看租户全部运行。列表支持按状态、工作流和时间筛选，并显示工作流、状态、进度、发起时间和完成时间。

管理员还可看到待审批和待人工介入项，但待办仍定位到所属运行与节点，不建立平行的业务状态源。

### 4.4 Agent 快捷入口

Agent 对话页可选择已发布工作流并跳转到专用执行页。对话页不承载工作流配置，也不复制运行详情能力。

## 5. 工作流生命周期

V1 生命周期为：

1. 创建草稿。
2. 显式保存草稿并增加修订号。
3. 验证最新草稿。
4. 发布不可变版本。
5. 在同一定义上继续编辑下一版草稿。
6. 只读查看历史版本。

发布操作必须二次确认并说明：发布会创建不可变版本，后续草稿修改不会改变已发布版本或历史运行。

V1 不包含删除、归档、回滚、复制、多人协作和外部触发器。

## 6. 输入模型

每个工作流定义由自然语言主任务和零到多个附加字段组成。主任务始终必填；管理员可配置附加字段的中文标签、说明、是否必填、默认值和约束。

V1 字段类型：

- 短文本
- 长文本
- 数字
- 单选
- 多选
- 布尔
- 日期

输入 schema 与 DAG 草稿一起保存，并在发布时冻结到版本。启动运行时，后端必须按目标版本的 schema 校验输入。字段错误返回可定位到字段名的结构化错误，且不得创建运行。

文件上传不属于 V1。

## 7. 执行与监控体验

### 7.1 启动前

执行页只展示工作流用途、当前发布版本、主任务、附加字段和开始按钮。技术参数不出现在普通用户首屏。提交后按钮进入 loading，成功创建运行后立即跳转到运行详情。

### 7.2 执行中

运行详情固定展示：

- 总体状态和完成节点数。
- 按运行快照渲染的 DAG。
- 节点状态颜色与当前节点。
- 当前节点的安全输出增量。
- 工具调用名称、耗时和摘要。
- 普通用户可用的取消操作。

SSE 使用单调事件序号恢复。浏览器刷新或连接短暂中断时显示“正在重新连接”，从最后事件序号续接；连接故障不等于运行失败。

### 7.3 完成、等待与失败

- 完成：展示最终结果，并以折叠步骤展示节点输出、工具调用和耗时。
- 等待审批：定位审批节点，普通用户看到等待原因，管理员看到处理入口。
- 人工介入：定位存在不确定外部副作用的节点，仅管理员执行允许的解决动作。
- 失败：失败节点标红并自动打开错误详情，保留已完成步骤与证据。
- 取消：展示取消原因和已完成步骤，不将取消显示为失败。

V1 不提供任意失败节点的手动重试。声明式自动重试继续由现有执行策略控制；检查点恢复与非幂等副作用重试需要单独设计。

## 8. 后端产品契约

### 8.1 数据变化

- Definition 和 Version 增加输入 schema；发布版本冻结 schema。
- Run 增加 `created_by`、`created_at`、`started_at`、`finished_at`。
- NodeAttempt 对外提供开始时间、结束时间和耗时。
- 工作流列表摘要提供草稿修订号、当前发布版本和更新时间。

### 8.2 查询能力

- 分页查询工作流，并按调用者角色过滤草稿或已发布项。
- 查询定义的版本列表与当前发布版本。
- 分页查询运行，并强制执行普通用户归属过滤。
- 按状态、工作流和时间筛选运行。
- 查询单次运行、节点尝试、审批、人工介入和可用操作。

### 8.3 实时事件

在现有持久事件与 SSE 游标上增加经过裁剪的节点输出增量和工具步骤事件。事件不得包含 bearer credential、API key、原始上游响应、敏感工具参数或 PII。输出事件用于展示，不替代最终持久化节点结果。

## 9. 冲突与错误处理

- 草稿修订冲突：返回冲突错误，前端保留本地内容并提示重新加载；禁止覆盖服务器草稿。
- 运行 generation 冲突：刷新运行与可用操作，说明状态已变化。
- 输入错误：显示字段级错误，不创建运行。
- 图验证错误：返回错误码、中文消息、节点或连线 ID。
- 资源失效：验证和发布失败关闭，不允许引用不存在或不可用的 Agent、Skill revision 或 MCP 工具。
- 权限变化：服务端拒绝后刷新权限并禁用操作。
- SSE 中断：重连并续传，不误报业务失败。
- 持久化失败：向上传播并展示失败，不返回伪成功。

## 10. 成功指标

- 管理员无需 JSON 即可完成包含全部五类节点的工作流创建与发布。
- 三节点顺序工作流首次创建并发布的可用性测试中位时间不超过 10 分钟。
- 普通用户在一个执行页面内完成输入并提交。
- 正常测试环境中，运行事件到前端可见延迟 P95 不超过 2 秒。
- 正常网络下刷新运行详情后 3 秒内恢复持久化状态。
- 端到端权限测试中跨用户读取和控制成功率为 0。
- 新增工作流业务逻辑与授权逻辑覆盖率不低于 80%。

时间指标必须绑定约定的测试环境；可用性指标需要人工验收，不能由单元测试代替。

## 11. 验证策略

- Domain/Application：输入 schema、列表过滤、对象授权、修订冲突、generation 冲突、事件续传。
- PostgreSQL：租户隔离、用户归属、分页稳定性、版本快照、事务回滚和历史 schema 升级。
- HTTP contract：新增集合接口、字段错误、结构化图验证错误和冻结响应兼容性。
- Frontend：表单生成、引导式画布、验证定位、状态映射、错误、空状态和权限入口。
- E2E：管理员创建并发布，普通用户启动并实时查看，管理员审批，运行完成。
- Failure E2E：节点失败、断线恢复、跨用户越权、并发编辑和外部副作用人工介入。
- Browser：桌面端完整流程；移动端覆盖发现、启动和运行查看，不要求画布编辑。

## 12. 证据与边界

仓库代码和测试证明执行内核已具备确定性 DAG、持久状态和控制点，但不证明列表、输入 schema、用户归属和输出增量已经实现。长期知识中已验证的观点支持把外层依赖、审批、恢复和权限放在确定性工作图中，把局部不确定性留在节点内部 Agent。React Flow 等具体画布库属于技术设计阶段选择，不在本产品设计中预先锁定。
