# Skill 草稿测试输出展示设计

## 目标

用户在 Skill 工作台的“测试”页签点击“运行草稿测试”后，能够明确看到执行状态、实际输出和必要的诊断信息，而不是只有输入 JSON 和短暂消息提示。

## 交互设计

- 未运行时不显示结果区。
- 运行中由按钮 loading 状态反馈，清除上一次结果。
- 执行成功后显示“执行结果”区域，包括成功状态、Trace ID、耗时和实际输出。
- 字符串输出按原文展示；对象、数组等结构化输出格式化为 JSON。
- 输出为 `null`、`undefined` 或空字符串时，显示“执行成功，但 Skill 未返回内容”。
- 执行失败时保留现有全局错误消息，同时在结果区显示失败状态和可理解的错误说明。

## 实现边界

- 仅调整 `SkillWorkspacePage` 的草稿测试状态和展示，不修改后端响应契约。
- 保留现有 `skillApi.testDraft` 和 `SkillTestResult` 类型。
- 复用 Ant Design 的 `Alert`、`Typography` 等现有组件，不引入新依赖。

## 数据流

1. 页面将输入解析后调用 `skillApi.testDraft`。
2. 成功响应完整保存在页面状态中，不再只保存 `result` 字段。
3. 结果组件根据响应渲染输出、Trace ID 和耗时。
4. 请求异常转换为用户可读文本，写入失败结果状态并通过 `message.error` 提示。

## 验证

- 组件测试验证字符串或结构化输出可见。
- 组件测试验证 Trace ID 和耗时可见。
- 组件测试验证空输出说明可见。
- 组件测试验证请求失败时页面内错误说明可见。
- 浏览器端执行真实草稿 Skill，确认输出在测试页签内展示。
