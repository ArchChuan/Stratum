# Stratum 源码细节面试题库扩展设计

## 目标

在现有 `tmp/agent-engineer-interview-report.md` 基础上，补充面向中高级 Agent 开发工程师的源码细节题库。题目同时覆盖 Go/HTTP 工程基础和 Stratum Agent 平台核心链路，并按实际面试热度排序。

## 内容范围

题库分为五层：

1. Go 与 HTTP 基础：JSON 解码、参数校验、错误处理、context、并发、接口、内存与性能。
2. 工程与架构：DDD、依赖注入、事务、多租户、缓存、消息、幂等和可观测性。
3. Agent 核心：ReAct、上下文工程、Skill、MCP、Memory、RAG、Workflow 和 Evaluation。
4. 稳定性与安全：超时预算、有限重试、降级、审批、权限交集、凭证保护和故障恢复。
5. 场景与改造题：定位现有实现、识别边界、提出最小修改并设计测试。

首批目标为 80 至 120 道高价值题。每题标记 `高频`、`进阶` 或 `专家`，并给出建议掌握深度。

## 单题结构

每道重点题采用统一结构：

- 问题；
- 30 秒回答；
- Stratum 源码定位与调用链；
- 底层原理；
- 容易答错的点；
- 连续追问及参考答案；
- 测试或运行证据。

简单事实题允许压缩为表格，但不得省略代码证据。通用框架行为与 Stratum 当前实现必须分开陈述。

## 证据规则

- 仓库代码、测试、ADR 和运行结果是项目事实源。
- Agent、Memory、Workflow、安全和数据治理题同时检索 Obsidian 已验证知识，并复核官方文档、标准、论文或上游源码。
- 不把搜索摘要、自动报告或 provisional 笔记当作关键证据。
- 每个具体结论至少给出源码路径；涉及行为契约时优先补充测试路径。
- 当前实现没有证明的能力必须明确写成边界或改进项，不能包装成已实现能力。

## 第一条示范题链

以“JSON 必填字段缺失如何校验”为示范，连续覆盖：

- `binding:"required"`、`ShouldBindJSON` 和 validator 的调用关系；
- 缺失、`null`、空字符串和 Go 零值的区别；
- 数值零值与 `required` 的冲突；
- PATCH 中使用指针或 nullable 类型表达字段是否出现；
- 嵌套结构与切片的 `dive` 校验；
- 未知字段、重复字段和请求体大小限制；
- 语法错误、绑定错误、校验错误与业务错误的 HTTP 映射；
- handler、middleware 和 contract test 的验证方式。

## 文档组织

保留现有报告章节，在“连续追问题库”之后新增“源码细节面试题库”。正文按五层分类，末尾增加高频速查表和源码索引。避免重复已有架构介绍；已有内容通过交叉引用连接。

## 验收标准

- 覆盖全部五层，首批不少于 80 道题。
- JSON 校验示范题链具备可直接用于模拟面试的完整答案。
- 高频题均有仓库代码路径，关键行为有测试证据。
- 答案区分通用原理、当前实现和改进建议。
- 文件内路径、符号和测试名经当前基线复核。
- 不虚构生产规模、性能指标、SLA 或未实现能力。
