# PRD: Agent 平台全量数据持久化系统

## 介绍

ClawHermes AI 平台需要一个企业级的数据持久化系统，用于保存 Agent 平台的全部运行状态。当前系统的 Agent、Skill、执行历史、内存和对话记录都存储在内存中，导致服务重启后数据丢失。本 PRD 定义了一个分层持久化架构，根据数据类型自动选择最优存储方案，支持完整的审计追溯、版本控制和企业级容错能力。

## 目标

- 实现全量数据持久化（Agent 配置、Skill 配置、挂载关系、执行历史、内存、对话记录）
- 优先实现关键数据（Agent/Skill 配置、挂载关系），后续补充其他数据
- 根据数据特性自动选择存储方案（Milvus、NATS、PostgreSQL、文件系统）
- 提供完整的审计日志和版本控制能力
- 支持数据恢复、中断点恢复和故障转移
- 实现企业级监控、告警和性能优化

## 非目标（超出范围）

- 不支持跨地域复制（多数据中心）
- 不支持实时数据同步到第三方系统
- 不支持自定义存储后端（仅支持预定义的四种）
- 不支持数据加密（由基础设施层处理）
- 不支持 GDPR 数据删除（仅支持软删除）

## 设计考虑

### 数据分层架构

```
┌─────────────────────────────────────┐
│   Application Layer (Agent/Skill)   │
├─────────────────────────────────────┤
│   Persistence Manager (统一接口)     │
├─────────────────────────────────────┤
│  Strategy Selector (自动路由)        │
├──────────┬──────────┬──────────┬────┤
│PostgreSQL│ Milvus  │  NATS    │ FS │
└──────────┴──────────┴──────────┴────┘
```

### 存储选择矩阵

| 数据类型 | 特性 | 存储方案 | 原因 |
|---------|------|--------|------|
| Agent 配置 | 结构化、需要事务 | PostgreSQL | ACID 保证 |
| Skill 配置 | 结构化、需要事务 | PostgreSQL | ACID 保证 |
| 挂载关系 | 向量化、快速查询 | Milvus | 向量相似度查询 |
| 执行历史 | 事件流、审计 | NATS | 事件溯源 |
| 内存数据 | 向量、相似度查询 | Milvus | 向量存储 |
| 对话记录 | 结构化、需要查询 | PostgreSQL | 灵活查询 |
| 审计日志 | 事件流、追溯 | NATS + PostgreSQL | 双写保证 |

## 技术考虑

### 依赖关系
- PostgreSQL 13+ (结构化数据)
- Milvus 2.0+ (向量数据)
- NATS 2.8+ (事件溯源)
- 现有的 Zap 日志系统

### 性能要求
- 单次读取延迟 <100ms
- 单次写入延迟 <200ms
- 批量操作吞吐量 >1000 ops/s
- 缓存命中率 >80%

### 容错要求
- 单个存储后端故障不影响整体可用性
- 支持自动故障转移
- 支持数据一致性检查和修复

### 扩展性
- 支持水平扩展（分片）
- 支持垂直扩展（更大的实例）
- 支持多租户隔离

## 成功指标

- 所有关键数据在服务重启后完整恢复
- 审计日志完整性 100%
- 数据一致性检查通过率 >99.9%
- 持久化操作延迟 <200ms（P99）
- 系统可用性 >99.5%
- 数据恢复时间 <5 分钟
